Interfaces de Usuario
Clientes externos · Web · Mobile · Integraciones
CLIENTES ▼

Cualquier aplicación, navegador o herramienta que quiera usar la plataforma lo hace a través de esta puerta de entrada.

Integraciones con sistemas externos: bancos, proveedores, plataformas de terceros
solicitudes HTTP · respuestas JSON
🚀
API
SolarisPlatform.API · Controllers · Middleware · JWT
.NET 10 ▼

El punto de entrada oficial. Recibe todas las peticiones, verifica identidad con JWT y las dirige al lugar correcto. Ninguna operación ocurre sin pasar primero por aquí.

Controllers: Auth, Usuarios, Roles, Empresas, Proyectos (5 sub-controllers), RRHH, Catálogos
Middleware: ExceptionHandlingMiddleware — captura errores y devuelve respuestas limpias
wwwroot: DevTools estáticos (index, swagger, test, health, api.html)
comandos y consultas validadas
⚙️
Aplicación
SolarisPlatform.Application · Services · DTOs · Validators
LÓGICA ▼

Las reglas de negocio: qué puede hacer cada usuario, cómo se calcula la nómina, qué sucede al cerrar un proyecto. Coordina el trabajo sin hablar directamente con la base de datos.

DTOs: Auth, Catalogos, Proyectos, Roles, RRHH, Usuarios — contratos de datos por módulo
Validators: FluentValidation — AuthValidators, CatalogoValidators, RrhhValidators
Mappings: AutoMapper profiles por módulo (Catálogos, Proyectos, RRHH)
objetos de dominio
🧩
Dominio
SolarisPlatform.Domain · Entities · Enums · Interfaces
NÚCLEO ▼

El corazón del sistema. La única capa que no depende de ninguna otra — es la base inmutable sobre la que todo lo demás se construye.

Entities: Seguridad (Usuario, RolPermiso), Empresas, Proyectos (7 archivos), RRHH, Catálogos
Interfaces/Repositories: IRepository, IRepositories, IProyectosRepositories, IRrhhRepositories
Enums: Enums generales, ProyectosEnums, RrhhEnums — tipado fuerte en todo el sistema
implementaciones concretas
🏗️
Infraestructura
SolarisPlatform.Infrastructure · Repositories · Services · EF Core
DATOS ▼

El puente entre el sistema y el mundo exterior. Implementa todo lo que el Dominio necesita. Si se cambia de base de datos o proveedor de correo, solo esta capa se toca.

Repositories: Seguridad, Catálogos, Proyectos, RRHH — patrón Repository + Unit of Work
Services: AuthService, RolService, UsuarioService, TokenPasswordServices + módulos especializados
RRHH Services: Asistencia, Capacitación, ConceptoNómina, Departamento, Evaluación, Préstamo, Puesto, Dashboard
SQL · Entity Framework Core
